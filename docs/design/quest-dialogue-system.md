**Key design changes (2026-03-07):**
- 3-layer quest system: Main Quest (14 Grovekeepers), World Quests (seed-variant narrative), Procedural Quests (65+ templates)
- 10 named NPCs are Tutorial Village residents; procedural NPCs populate the infinite world
- Relationship system with 4 tiers (Stranger/Acquaintance/Friend/Best Friend)
- NPCs use ChibiCharacter GLBs with seeded appearance + anime.js rigid-body animation
- World quests use A/B/C dialogue variations seeded from world seed

## 9. 3-Layer Quest Architecture

The unified design defines three quest layers that work together:

### Layer 1: Main Quest -- The Grovekeeper Path

Find all 14 dormant Grovekeepers sleeping in hedge labyrinths across the infinite
world. Non-linear (any order except Worldroot last). Each Grovekeeper encounter
unlocks a tree species and reveals lore.

- Compass hints guide toward undiscovered labyrinths
- Each labyrinth is biome-appropriate (Willowsong in Wetlands, Frostbark in Frozen Peaks)
- Difficulty escalates: 8x8 (easy) to 20x20 (Worldroot final)
- Finding all 14 + Worldroot = "You are now a Grovekeeper" achievement, NG+ unlock

### Layer 2: World Quests (Seed-Variant Narrative)

The overarching narrative spine. Each seed generates a unique world quest chain:

```
scopedRNG('world-quest', worldSeed) -> questTemplateId -> A/B/C dialogue variant
```

Examples:
- "The Withered Road" -- a blight spreads; find its source (variant: mushroom/frost/shadow)
- "The Lost Village" -- ruins of a settlement; piece together what happened
- "The Merchant's Burden" -- Hazel asks for help with a dangerous trade route

Each world quest spans 3-5 steps across multiple chunks, guiding the player toward
undiscovered areas. Different seeds produce different narrative arcs.

### Layer 3: Procedural Quests

Generated by village NPCs from templates + seeded context:

- **65+ templates** across 9 categories: gather, plant, explore, deliver, build, discover, hunt, fish, craft
- **Daily quests:** 1-4 per day scaling by level, streak bonuses at 3/5/7/10/14 days
- **Context-sensitive:** templates pick resources/species/locations near the NPC's village
- **Relationship-gated:** better quests from higher-relationship NPCs
- **Rewards scale** with distance from origin (farther = better)

## 2. Daily Quest System

### Overview

Daily quests provide short-session goals that refresh every in-game day. They complement the longer quest chains by giving immediate, achievable objectives.

### Quest Count by Level

| Player Level | Easy | Medium | Hard | Epic | Total |
|-------------|------|--------|------|------|-------|
| 1-2 | 1 | 0 | 0 | 0 | 1 |
| 3-6 | 1 | 1 | 0 | 0 | 2 |
| 7-14 | 1 | 1 | 1 | 0 | 3 |
| 15+ | 1 | 1 | 1 | 1 | 4 |

This scaling is already implemented in `generateDailyQuests()` in `game/systems/quests.ts`.

### Generation Algorithm

The existing implementation uses seeded RNG. The enhanced algorithm adds weighting by recent activity and current season:

```typescript
function generateDailyQuests(
  currentSeason: Season,
  playerLevel: number,
  completedGoalIds: Set<string>,
  worldSeed: string,
  recentActivity?: RecentActivity, // NEW: weight based on what player has been doing
): ActiveQuest[] {
  // Existing logic generates by difficulty tier.
  // Enhancement: weight category selection by what player HASN'T done recently.

  // If player has been planting a lot, weight toward harvesting/collection.
  // If player hasn't explored, weight toward exploration.
  // If current season matches a seasonal goal, boost seasonal weight.

  const categoryWeights = computeCategoryWeights(recentActivity, currentSeason);
  // ... use weights in selectRandomGoals
}

interface RecentActivity {
  treesPlantedLast3Days: number;
  treesHarvestedLast3Days: number;
  treesWateredLast3Days: number;
  zonesVisitedLast3Days: number;
  resourcesCollectedLast3Days: number;
}

function computeCategoryWeights(
  activity: RecentActivity | undefined,
  season: Season,
): Record<GoalCategory, number> {
  const base: Record<GoalCategory, number> = {
    planting: 1.0,
    harvesting: 1.0,
    watering: 1.0,
    growth: 1.0,
    collection: 1.0,
    exploration: 1.0,
    seasonal: 1.0,
    economic: 1.0,
    mastery: 0.5, // mastery quests are rarer as dailies
  };

  if (!activity) return base;

  // Inverse weighting: categories the player hasn't engaged with get boosted
  if (activity.treesPlantedLast3Days > 10) {
    base.planting *= 0.5;
    base.harvesting *= 1.5;
  }
  if (activity.treesHarvestedLast3Days < 2) {
    base.harvesting *= 1.5;
  }
  if (activity.zonesVisitedLast3Days < 1) {
    base.exploration *= 2.0;
  }

  // Season boost
  base.seasonal *= (season === "spring" || season === "autumn") ? 1.5 : 1.0;

  return base;
}
```

### Mood System

NPC mood is computed each time the player interacts, based on game state:

```typescript
type NpcMood = "happy" | "neutral" | "worried";

interface MoodCondition {
  type: "resource_low" | "resource_high" | "weather" | "tree_count"
       | "quest_complete" | "species_diverse" | "season";
  parameter?: string;
  threshold?: number;
}

function computeNpcMood(
  npcId: string,
  gameState: GameStateData,
  weather: WeatherState,
): NpcMood {
  const moodRules = NPC_MOOD_RULES[npcId];
  if (!moodRules) return "neutral";

  // Check worried conditions first
  for (const condition of moodRules.worried) {
    if (evaluateMoodCondition(condition, gameState, weather)) {
      return "worried";
    }
  }

  // Check happy conditions
  for (const condition of moodRules.happy) {
    if (evaluateMoodCondition(condition, gameState, weather)) {
      return "happy";
    }
  }

  return "neutral";
}
```

Mood affects which greeting line variant is selected, but does NOT gate dialogue options. This is a cozy game -- moods add flavor, not barriers.

## 4. Dialogue Tree Format

### Schema

The existing `DialogueNode` schema in `game/npcs/types.ts` is sufficient for the current feature set. The enhanced schema adds conditions, mood variants, and relationship gating:

```typescript
/** Enhanced dialogue node -- backwards-compatible with existing schema. */
interface DialogueNodeV2 extends DialogueNode {
  /** Optional conditions that must be met to show this node. */
  conditions?: DialogueCondition[];

  /** Optional mood-variant overrides for the text. */
  moodVariants?: {
    happy?: string;
    worried?: string;
  };

  /** Minimum relationship level required to access this node. */
  minRelationship?: RelationshipTier;
}

type RelationshipTier = "stranger" | "acquaintance" | "friend" | "bestFriend";

interface DialogueCondition {
  type: "min_level" | "has_quest" | "has_achievement"
      | "season" | "min_relationship" | "chain_completed"
      | "weather" | "has_species" | "time_of_day";
  value: string | number;
}

interface DialogueChoiceV2 extends DialogueChoice {
  /** Conditions to show this choice. Hidden if not met. */
  conditions?: DialogueCondition[];

  /** Friendship points gained by selecting this choice. */
  friendshipPoints?: number;
}
```

### Player Choice Format

- **Maximum 4 choices** per node (mobile-friendly: 4 buttons fit in portrait mode at 44px touch targets)
- **Minimum 1 choice** (even "Goodbye" counts)
- **Recommended: 2-3 choices** for most nodes -- one progresses the topic, one asks a follow-up, one ends the conversation
- **"Goodbye" choice** always present as the last option, typically awards a small XP bonus

### Consequence Tracking

Dialogue choices affect three things:

1. **Friendship Points:** Most choices award 1-3 friendship points with the speaking NPC.
2. **NPC Mood (Indirect):** Completing quests from dialogue raises friendship, which influences future greetings. But mood is computed from game state, not stored dialogue history.
3. **Future Dialogue Availability:** High-friendship dialogue nodes gate behind `minRelationship`. Completed quest chains unlock new dialogue branches via `chain_completed` conditions.

Choices do NOT have negative consequences. This is a cozy game. No choices lock out content permanently. Higher friendship simply unlocks additional dialogue branches.

## 7. Relationship System

### Friendship Levels

| Points | Tier | Label | Visual |
|--------|------|-------|--------|
| 0-9 | 0 | Stranger | No heart icon |
| 10-29 | 1 | Acquaintance | 1 grey heart |
| 30-59 | 2 | Friend | 2 green hearts |
| 60-100 | 3 | Best Friend | 3 gold hearts |

### Gaining Friendship

| Action | Points | Limit |
|--------|--------|-------|
| First conversation | 5 | Once per NPC |
| Complete their quest chain step | 3-15 | Per step (defined in `friendshipPoints` reward field) |
| Complete their full quest chain | 10 bonus | Once per chain |
| Trade with them (Hazel) | 1 per trade | Max 3 per day |
| Buy seeds (Blossom) | 1 per purchase | Max 3 per day |
| Craft at their workshop (Oakley) | 1 per craft | Max 3 per day |
| Gift a rare resource | 2-5 | Max 1 per day |
| Talk to them (any dialogue) | 1 | Max 2 per day |
| Complete a daily quest | 1 to a random NPC | Once per quest |

### Friendship Rewards

| Tier | Reward |
|------|--------|
| Acquaintance (10+) | NPC greets by name. Unlock second dialogue branch. |
| Friend (30+) | NPC offers 10% better trade rates. Unlock friendship-gated dialogue. NPC gives a small gift on first reaching this tier (5 of their specialty resource). |
| Best Friend (60+) | NPC offers 20% better trade rates. Unlock exclusive dialogue revealing deep lore/backstory. NPC gives a rare seed on first reaching this tier. Exclusive quest chain step unlocks (the final "best friend" step in their chain). |
